# 航空工具箱微信小程序设计

## 背景

航空工具箱是一款面向航空出行和航空信息查询场景的微信小程序。第一版不接入航班动态数据，专注提供航空相关静态信息查询和常用辅助工具。

小程序采用原生微信小程序与微信云开发实现。数据以云数据库为主，本地静态数据为兜底，确保在云环境未配置或云端加载失败时仍可演示和测试核心功能。

## 第一版范围

第一版包含以下功能：

- 航空公司客服电话查询。
- 全国机场客服电话查询。
- 航空公司两字代码查询。
- 全国机场三字代码查询。
- 航空常用城市时间查询。
- 常用币种汇率换算演示。

第一版不包含航班动态、机票价格、航班延误预测、实时汇率接口、用户登录、收藏同步和后台管理系统。

## 技术路线

采用原生微信小程序、微信云开发和静态数据集合。

项目结构建议：

- `miniprogram/`：小程序前端，包含页面、公共组件、样式、工具函数和本地兜底数据。
- `cloudfunctions/`：云函数，第一版提供统一静态数据查询函数，后续可扩展汇率实时查询函数。
- `cloudbase/seed/`：云数据库初始化 JSON 数据。
- `docs/`：项目说明、云开发部署说明和数据维护说明。

选择原生微信小程序的原因：

- 更贴近微信开发者工具和云开发默认工作流。
- 不引入额外组件库，降低依赖和审核风险。
- 第一版以查询工具为主，原生组件足够支撑体验。

## 页面结构

首页直接展示五个工具入口，不做营销型欢迎页。

页面包括：

- 首页：展示客服电话查询、航司代码、机场代码、城市时间、汇率换算入口。
- 客服电话查询页：支持航空公司、机场名称、城市、代码搜索。
- 航空公司两字代码查询页：支持中文名、英文名、两字码搜索。
- 机场三字代码查询页：支持机场名、城市、三字码搜索。
- 航空常用城市时间页：展示常用城市当前时间，支持搜索。
- 汇率换算页：支持输入金额、选择原币种和目标币种，基于示例汇率实时计算。

## 数据模型

### `airlines`

航空公司基础信息集合。

- `nameCn`：中文名。
- `nameEn`：英文名。
- `iataCode`：航空公司两字码。
- `icaoCode`：航空公司三字代码，可选。
- `phone`：客服电话。
- `website`：官网，可选。
- `keywords`：搜索关键词数组。
- `status`：数据状态，取值为 `active` 或 `inactive`。

### `airports`

全国机场基础信息集合。

- `nameCn`：机场中文名。
- `city`：所在城市。
- `province`：省份、自治区或直辖市。
- `iataCode`：机场三字码。
- `icaoCode`：机场四字码，可选。
- `phone`：客服电话。
- `keywords`：搜索关键词数组。
- `status`：数据状态，取值为 `active` 或 `inactive`。

### `aviationCities`

航空常用城市时间集合。

- `cityCn`：城市中文名。
- `countryCn`：国家或地区。
- `timezone`：IANA 时区，例如 `Asia/Shanghai`。
- `utcOffset`：标准 UTC 偏移，例如 `+08:00`。
- `airportCodes`：关联机场三字码数组。
- `sortOrder`：展示排序。

第一版覆盖航空常用城市和机场所在城市，不追求全球所有城市完整覆盖。

### `currencies`

常用币种示例汇率集合。

- `code`：币种代码，例如 `CNY`。
- `nameCn`：币种中文名。
- `symbol`：币种符号。
- `sampleRateToCny`：相对人民币的示例汇率。
- `updatedAt`：示例数据更新时间。
- `isSample`：固定为 `true`。

汇率第一版用于界面和换算流程演示，不代表实时汇率。

### `appMeta`

应用元信息集合。

- `dataVersion`：数据版本。
- `updatedAt`：数据更新时间。
- `notice`：数据说明或免责声明。

## 查询规则

- 所有代码搜索大小写不敏感。
- 中文查询支持名称、城市、省份和关键词模糊匹配。
- 电话号码展示为可点击内容，调用 `wx.makePhoneCall`。
- 代码字段突出展示，便于快速识别和抄录。
- 空结果提示用户更换关键词，可输入名称、城市或代码。
- 所有静态信息页展示“信息可能变化，请以航空公司或机场官方渠道为准”。

## 云开发边界

第一版优先读云数据库。如果云环境未配置、网络失败或集合为空，小程序自动使用本地兜底数据。

云函数：

- `searchStaticData`：统一静态数据查询函数，参数包括 `type`、`keyword` 和 `limit`。

前端本地计算：

- 城市时间：云端只提供城市和时区配置，小程序端根据时区计算当前当地时间。
- 汇率换算：第一版基于示例汇率在小程序端计算。

预留能力：

- 后续可新增 `getExchangeRates` 云函数接入第三方实时汇率接口。
- 后续可增加后台数据维护入口或导入脚本，但不进入第一版范围。

## 视觉与文案

- App 内所有文字使用简体中文。
- 字体声明优先使用思源黑体：`"Source Han Sans SC", "Noto Sans CJK SC", sans-serif`。
- 微信端无法保证所有设备已安装思源黑体，因此通过系统字体回退保证可读性。
- 界面定位为工具型产品，信息密度适中，搜索优先。
- 主色建议使用航空蓝和中性灰，少量使用橙色强调代码、电话等关键信息。

## 错误状态

- 云数据加载失败：自动切换到本地兜底数据，并提示“当前使用内置数据”。
- 无搜索结果：提示“未找到相关数据，请尝试其他关键词”。
- 电话缺失：显示“暂无客服电话”。
- 汇率数据为示例：显示“示例汇率，仅供功能演示”。
- 汇率示例数据有更新时间，过期仍可计算但不声称实时准确。

## 测试与验收

第一版交付标准：

- 能在微信开发者工具中打开并运行。
- 五个工具入口全部可用。
- 航空公司、机场、城市时间和币种数据均可查询或演示。
- 云开发初始化代码和云函数结构完整。
- 未配置云环境时仍可通过本地兜底数据演示。
- 提供数据维护说明，说明如何更新 JSON 并导入云数据库。
- 提供基础测试覆盖搜索匹配、时间格式化、汇率计算和数据字段完整性。

## 数据维护原则

航空公司、机场客服电话、代码和城市时区数据以结构化 JSON 维护。每次更新应同步修改数据版本和更新时间。

数据内容可能随官方渠道变化。产品应持续提示用户以航空公司、机场或相关官方渠道为准。
